The C++ template feature allows for generic programming by defining families of classes, functions, or aliases that can operate with arbitrary types and values. Templates are parameterized by one or more template parameters, which can be types, constants, or template templates. They enable the creation of flexible and reusable code components without sacrificing performance.

### Core Concept
- **Templates** define generic entities (classes, functions, etc.) that can operate with any type.
- **Template Parameters** include types, non-type parameters (constants), and template templates.
- **Specialization** allows for specific implementations of templates tailored to particular types or values.
- **Constraints** ensure that the template's requirements are met; they can be expressed using concepts introduced in C++20.

### Types/Constraints
- **Type Parameters**: Define a family of types (e.g., `template <typename T> class Container { ... }`).
- **Non-type Parameters**: Fixed values that must be known at compile time, e.g., integer or enum values (`template <int N> struct Array { ... }`).
- **Concepts** in C++20 provide a way to constrain template parameters with logical conditions (e.g., `concept C = sizeof(T) != sizeof(int);`).

### Key APIs
- **Template Declaration**: Defines the template without providing an implementation (`template <typename T> void func() { ... }`).
- **Template Definition**: Provides the actual implementation of a templated entity (`template <typename T> void func() { ... }`).
- **Specialization**: Provides specific implementations for particular types or values (`template <> void func<int>() { ... }`).

### Tricky Edge Cases
- **Substitution Failure Is Not An Error (SFINAE)** can be used to handle cases where template arguments do not match the expected type.
- **Partial Specialization**: Allows for more than one implementation of a template based on partial information about its parameters (`template <typename T> class Container;`).
- **Dependent Names**: When names depend on template parameters, requiring careful handling in implementations to avoid errors or undefined behavior.

### Typical Usage Patterns
1. **Generic Containers and Algorithms**: Define containers like `std::vector<T>` and algorithms that can operate on any type (`template <typename T> void sort(Container<T>&)`).
2. **Metaprogramming**: Use templates to perform compile-time computations, such as determining the size of a container at compile time (`template <typename T, std::size_t N> struct array_size { static constexpr std::size_t value = N; };`).
3. **Functional Programming Constructs**: Implement higher-order functions that can work with any callable object (`template <typename F, typename... Args> auto apply(F&& f, Args&&... args)`).

### Example Program
```cpp
// Generic function to print elements of a container
template <typename T>
void print_container(const T& container) {
    for (const auto& item : container) {
        std::cout << item << " ";
    }
    std::cout << std::endl;
}

int main() {
    // Usage of the generic function with different containers
    std::vector<int> vec = {1, 2, 3};
    print_container(vec);

    std::list<double> lst = {1.1, 2.2, 3.3};
    print_container(lst);

    return 0;
}
```
This example demonstrates the use of a template function to handle different types of containers uniformly, showcasing its flexibility and reusability in C++.
